# PAP CheckList - End User Documentation Plan

## Overview

This document outlines the documentation to be created for end users of the PAP CheckList application.

---

## 1. Getting Started Guide

### Purpose
Help new users quickly understand and start using the application.

### Contents
- **Introduction**: What is PAP CheckList and who is it for
- **System Requirements**: Browser compatibility, screen size recommendations
- **First Login**: Accessing the application
- **Quick Start**: Create your first checklist in 5 minutes

---

## 2. User Guide

### Purpose
Comprehensive reference for all application features.

### Chapters

#### 2.1 Dashboard
- Viewing your checklists
- Understanding status indicators (Draft, In Review, Final)
- Creating a new checklist
- Opening an existing checklist
- Searching and filtering checklists

#### 2.2 Checklist Editor
- **Document Layout**: Understanding workgroups and sections
- **Workgroups**: Navigating numbered sections (20, 40, 180, etc.)
- **Checklist Rows**: Adding, editing, and removing items
- **Answer States**: Understanding the 7 status options
  - YES (Included)
  - NO (Excluded)
  - BLANK (Intentionally unanswered)
  - PS (Provisional Sum)
  - PC (Prime Cost)
  - SUB (Subcontractor)
  - OTS (Owner to Supply)
- **Notes**: Writing detailed assumptions and explanations
- **Auto-Save**: How your work is automatically saved

#### 2.3 Adding Images
- Pasting screenshots from clipboard
- Drag and drop image upload
- Adding captions to images
- Removing images

#### 2.4 Revisions
- What are revisions and why use them
- Creating a new revision
- Writing revision summaries
- Viewing historical snapshots
- Understanding read-only mode for revisions

#### 2.5 Branding & Settings
- **Client Logo**: Uploading a custom logo for reports (via Branding tab)
- **Preview**: Verifying how the logo appears
- **Fallback**: Understanding default branding (PAP Logo)

#### 2.6 Exporting to PDF
- **Generating**: How to create the report
- **Progress**: Understanding the generation process steps & progress bar (e.g. "Processing Images...")
- **Cancelability**: How to stop generation if needed
- **Upload**: Where to find the saved PDF copies (SharePoint)
- **What's included**: Workgroups, Notes, Images (Optimized High-Res)

## Writing Style Guidelines

1. **Use simple language** - Avoid technical jargon
2. **Be task-focused** - "How to..." format
3. **Include screenshots** - Show don't just tell
4. **Use bullet points** - Easy to scan
5. **Keep paragraphs short** - 2-3 sentences max
6. **Highlight important info** - Use callouts for tips and warnings
